# 数据同步使用指南

## 📚 概述

AI Hub 现已支持从 GitHub 仓库自动同步数据。本指南将帮助您了解如何使用数据同步功能。

## 🔄 工作原理

```
GitHub Repo (ai-hub-data)
         ↓
GitHub API (首次启动 + 手动同步)
         ↓
前端应用 → localStorage (本地缓存)
         ↓
    用户数据优先 + 自动补充新数据
```

### 数据来源

所有数据来自公开的 GitHub 仓库：
- **仓库地址**: https://github.com/snipercai/ai-hub-data
- **数据格式**: Markdown with Front Matter + Embedded JSON
- **数据类型**: 行业资讯、AI 工具、提示词、项目案例、学习资源、学习记录

## ✨ 主要功能

### 1. 首次启动自动同步

当您首次启动应用时，系统会自动从 GitHub 仓库拉取最新数据到本地。

**特点**:
- ✅ 无需手动操作
- ✅ 自动检测是否首次启动
- ✅ 失败不影响应用正常使用

### 2. 手动同步

您可以随时通过管理后台手动触发数据同步。

**操作路径**: 
1. 访问管理后台 `/admin`
2. 点击侧边栏的"数据同步"菜单
3. 点击"同步数据"按钮

### 3. 数据保护

同步时采用"用户数据优先"策略:
- ✅ 您本地的所有数据都会被保留
- ✅ GitHub 数据只补充新条目
- ✅ 不会覆盖您已有的数据
- ✅ 支持离线使用

## 📋 使用步骤

### 首次使用

1. **打开应用** - 首次启动时会自动同步数据
2. **等待完成** - 同步过程约需 3-10 秒
3. **开始使用** - 同步完成后即可使用最新数据

### 手动同步

1. **进入管理后台** - 访问 `/admin`
2. **选择数据同步** - 点击侧边栏"数据同步"菜单
3. **查看状态** - 查看最后同步时间
4. **触发同步** - 点击"同步数据"按钮
5. **等待完成** - 同步完成后会显示结果

### 查看同步状态

在数据同步页面，您可以看到:
- ✅ 最后同步时间
- ✅ 同步按钮状态（同步中/可同步）
- ✅ 同步说明和帮助信息

## 🔧 技术细节

### 数据格式

GitHub 仓库中的数据采用 Markdown 格式存储:

```markdown
---
id: news-1
title: OpenAI 发布 GPT-5
category: 技术
tags: ["AI", "GPT", "技术"]
updatedAt: 2024-04-17T10:00:00Z
---

# Markdown 内容

<!-- DATA_JSON_START
{"id": "news-1", "title": "..."}
DATA_JSON_END -->
```

### 版本控制

每次数据更新都会更新 `version.json`:

```json
{
  "version": "1.0.0",
  "updatedAt": "2024-04-17T10:00:00Z",
  "changelog": "初始版本",
  "dataCount": {
    "news": 2,
    "tools": 2
  }
}
```

### 本地缓存

数据缓存在浏览器的 localStorage 中:
- Key 格式：`ai-hub-{category}`
- 版本信息：`ai-hub-local-version`
- 同步时间：`ai-hub-last-sync-time`

## ❓ 常见问题

### Q1: 同步失败怎么办？

**A**: 同步失败不会影响您的本地数据。可能的原因:
- 网络连接问题
- GitHub API 限流（60 次/小时）
- 仓库暂时不可用

**解决方案**:
1. 检查网络连接
2. 稍后重试
3. 如持续失败，可联系管理员

### Q2: 我的数据会丢失吗？

**A**: 不会！同步采用"用户数据优先"策略:
- 您添加的数据会保留
- 您修改的数据会保留
- GitHub 数据只补充新条目

### Q3: 如何查看数据源？

**A**: 访问 GitHub 仓库查看:
- 仓库首页：https://github.com/snipercai/ai-hub-data
- 数据目录：`/news`, `/tools`, `/prompts` 等
- 版本文件：`version.json`

### Q4: 可以贡献数据吗？

**A**: 当然可以！欢迎通过以下方式贡献:
1. 在 GitHub 仓库提交 Issue
2. 提交 Pull Request
3. 联系项目维护者

### Q5: 多久同步一次？

**A**: 建议:
- 首次启动：自动同步
- 之后：根据需要手动同步
- 推荐：每周或每月同步一次
